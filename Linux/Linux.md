# Linux Fundamentals

- Users (Application) talk to Operating System so that OS can interact with the server.
- OS is a bridge between software and hardware or acts as a bridge between user and hardware.
- OS performs process management, memory management, device management and network managment.

## OS
- An Operating System (OS) is system software that acts as an interface between a computer's hardware and the user or application programs. It's basically the backbone of any computer system.

What does an Operating System do?
Here are the main functions and uses:

🔧 1. Resource Management
Manages hardware components like CPU, memory, disk, and input/output devices.

Decides which process gets how much of each resource and when.

👥 2. User Interface
Provides a way for users to interact with the computer (like graphical UI or command-line interface).

Examples: Windows Desktop, Linux Terminal.

🧠 3. Process Management
Handles running applications (called processes).

Manages multitasking—deciding which process runs at what time.

💾 4. Memory Management
Keeps track of each byte in a computer's memory.

Allocates memory to programs when needed and frees it when not.

📂 5. File System Management
Organizes and manages data storage in files and directories.

Provides access controls and handles reading/writing from storage.

🌐 6. Device Management
Manages all input/output devices via drivers.

Allows communication between software and hardware (like printers, keyboards, etc).

🔐 7. Security and Access Control
Protects system data and resources from unauthorized access.

Manages user logins and permissions.
